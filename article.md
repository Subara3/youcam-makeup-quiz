## 同じシーンに、違う差分

同じ会議室、同じ人。違うのは今日の気分だけ。それをリップの色温度、チークの有無、眉の強さで描き分けているつもりですが、さて、どうでしょうか。ピンとくるでしょうか。よーくよーく見てみるとメイクが少しずつ違うことに気が付きますね。

## どんな色で塗り分けたか

気分ごとに、リップを中心に色と質感を決めています。だいたいこんな配合です。

| 気分 | リップ | 目もと・頬 |
| --- | --- | --- |
| はしゃぎ | ピンクのグロス `#e49aa6` | 頬にピンク、ラメのアイシャドウ、ツヤ多め |
| いらだち | 深い赤のマット `#7d2c38` | 眉とアイラインを締める、スモーキー |
| しょんぼり | くすんだ色のマット `#a87b78` | 頬は置かず、ツヤ控えめ |
| ごきげん | コーラルのシアー `#e2785d` | 自然な血色、軽いツヤ |

色は「この赤にしたい」という狙いの値で、実際には YouCam のいちばん近い色を選んで乗せています。並べてみると、リップの色温度が少しずつ違うのが分かるでしょうか。逆に、濃さはそこまで強くないので、本当にうっすらした差です。そこがまた「言われないと気づかない」感じで、このクイズに合っている気がしています。

## 「いちばん近い色」をどう選ぶか

ひとつ、地味だけど気に入っている処理があります。

YouCam のメイクは、あらかじめ並んだ色玉（パレット）から選ぶ方式です。でも僕が決めた色（たとえば「いらだち」の `#7d2c38`）が、パレットにそのまま同じ色で存在するとは限りません。近い色はあっても、ぴったりは無い。

そこで、**パレットの色玉を全部読み取って、狙った色にいちばん近い玉を選ぶ**ようにしました。色玉はそれぞれ自分の色を `background-color` として持っているので、そこから RGB を数値で取り出せます。

```js
// 狙った色 (tr, tg, tb) に最も近い色玉を選ぶ
const balls = [...document.querySelectorAll('[class*="color-ball_singleColor"]')];

let best = null;
let min = Infinity;

for (const el of balls) {
  // 色玉の実際の色 "rgb(125, 44, 56)" などを数値の配列にする
  const [r, g, b] = getComputedStyle(el).backgroundColor.match(/\d+/g).map(Number);

  // 狙いの色との「距離」。RGB を座標とみなして二乗距離で測る
  const dist = (r - tr) ** 2 + (g - tg) ** 2 + (b - tb) ** 2;

  if (dist < min) {
    min = dist;
    best = el;
  }
}

best.click(); // いちばん近い色玉を選択
```

やっていることはシンプルで、RGB を 3 次元の座標とみなして、狙いの色との距離がいちばん短い玉を探しているだけです。

厳密にいうと、この「RGB の距離」は人の目が感じる色の近さとは少しズレます（同じ距離でも、緑のズレより青のズレのほうが気になりにくい、みたいな差があります）。ちゃんとやるなら一度 Lab という色空間に変換して距離を測るのが定番ですが、今回はパレットの色数も多く、ぴったりでなくても雰囲気が出れば良かったので、単純な RGB 距離で十分でした。

この「狙った色 → 最寄りの色玉」を、リップ・チーク・アイシャドウ……と決めた色の数だけ繰り返して、1 枚の顔を作っています。

## こんなのも面白そう、という話

作っているうちに、広げ方をいくつか思いつきました。

- **お題を当てる、人狼みたいな遊びにする**。「AIアートインポスター」のように、何人かが同じお題でメイクをして、ひとりだけ違うお題。どれが紛れ込んでいるかを当てる、という形にしても盛り上がりそうです。
- **もっと"伝わる"ようにする**。いまは差が繊細すぎるので、濃さまで指定できるAPIを使えば、狙った気分がもう少しはっきり顔に出るはずです。難易度を選べるようにするのも良さそう。
- **自分の顔で参加する**。自分の写真に「今日の気分メイク」を乗せて投稿し、みんなで当て合う。あるあるは人それぞれなので、お題はいくらでも増えていきます。

メイクって、こんなふうにデータとして遊べるんだ、というのが個人的な発見でした。
