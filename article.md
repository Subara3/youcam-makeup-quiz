## きっかけ

会社に、人の変化にやたら気づく人がいます。「髪切った？」だけじゃなくて、「アイシャドウ変えた？」まで当ててくる。すごいなあと思いつつ、自分はファンデもアイシャドウも、正直まったく分かりません。

その「言われないと分からないけど、本人の中では切実」な気分を、メイクに仕込んだら当てられるのか。たとえば会議中のこの顔。

> 微熱があるので場合によっては帰りたい

血色をひとつ抜いて、ツヤを消す。リップはくすませて、でも眉とアイラインは残す（まだ帰ると決めたわけじゃないので）。……たぶん伝わりません。そこが面白いだろう、という遊びです。

## 気分は、顔のどこに出る？

作るにあたって、まず「喜怒哀楽が顔のどこに出るか」を決めました。同じ人・同じ構図なので、ここがブレると全部同じ顔になってしまいます。

| 気分 | 主にどこへ | 具体的に |
| --- | --- | --- |
| 喜 | ツヤと輝き | ファンデのツヤを上げ、ラメのアイシャドウ、グロスのリップ、ハイライト強め |
| 怒 | 目元と口元の輪郭 | 眉とアイラインを締める、深い赤のマットリップ |
| 哀 | 肌の血色 | ファンデで血色を抜く（ツヤ最小）、くすんだ薄いリップ、他は最小限 |
| 楽 | やわらかさ | 自然な血色、シアーなコーラルリップ、尖った要素を入れない |

考え方としては、**色相で温度（暖色＝はしゃぎ／寒色寄りのくすみ＝しょんぼり）、質感で強さ（マット＝こわばり／グロス＝高揚）、血色（ファンデ・チーク・ハイライト）でエネルギーの量**を出しています。そして主役はリップ。顔の中で色がいちばん動いて見えるのが口元なので、感情の中心はだいたいそこに置いています。

おもしろいのは、**哀をいちばんはっきり作れたのがファンデ**だったこと。リップを変えるより、肌の血色とツヤを抜くほうが「だるい」が伝わります。

## YouCam の API でメイクを乗せる

差分は全部で56枚（14シーン × 4気分）。手作業はつらいので、YouCam の makeup-vto API で作りました。

認証は思ったよりあっさりで、APIキーを `Authorization: Bearer ...` に載せるだけ（V2）。流れは「画像をアップロード → エフェクトを指定してタスク実行 → 結果画像のURLを受け取る」の3ステップです。仕上がりは透かしなしの高解像度で返ってきます。

メイクは `effects` 配列で渡します。たとえばファンデとリップだけならこんな感じ。

```json
{
  "src_file_id": "（アップロードした画像のID）",
  "effects": [
    { "category": "foundation",
      "palettes": [{ "color": "#ece0da", "colorIntensity": 70,
                     "coverageIntensity": 75, "glowIntensity": 4,
                     "texture": "matte", "smoothness": 50, "thickness": 50 }] },
    { "category": "lip_color", "shape": { "name": "original" }, "style": { "type": "full" },
      "palettes": [{ "color": "#a87b78", "texture": "matte", "colorIntensity": 45 }] }
  ]
}
```

色は `#RRGGBB`、濃さは `colorIntensity`（0〜100）。これがちゃんと効くので、「哀のリップは濃さ45」みたいに弱さまで指定できます。Webのエディタだと濃さは触れなかったので、ここはAPIの良いところでした。

## カテゴリごとにスキーマが全然違う

ここが今回いちばん手こずったところです。メイクのカテゴリ（ファンデ、チーク、アイシャドウ、アイライナー、眉、ハイライト、リップ）で、**必要なフィールドがそれぞれ違います**。

最初は手探りでした。適当に投げると、バリデーションエラーが「このカテゴリにはこのフィールドが要る」と少しずつ教えてくれるので、それを頼りに1カテゴリずつ詰めていった感じです。分かったことの一部：

- リップは `shape` と `style.type`（`full`/`ombre`/`twoTone`）が必要。グロスなら `gloss` と `transparencyIntensity`、シアーなら `transparencyIntensity` も足す。
- アイシャドウ・チーク・アイライナー・眉・ハイライトは、**`pattern`（メイクの“型”）の指定が必須**。しかもその `pattern.name` は適当な文字列だと弾かれて、実在するパターンID（ラベル）でないと通りません。
- ラベルは公開カタログで分かりました（例：`https://plugins-media.makeupar.com/wcm-saas/patterns/eyeshadow.json`）。`1color1` のような名前で、`1 color`／`2 colors` の別はそのまま「色（パレット）をいくつ渡すか」に対応します。

たとえばラメのアイシャドウはこう。

```json
{ "category": "eye_shadow",
  "pattern": { "type": "color", "name": "1color1" },
  "palettes": [{ "color": "#caa15a", "texture": "shimmer", "colorIntensity": 55,
                 "shimmerColor": "#f5e3a0", "shimmerDensity": 40,
                 "shimmerIntensity": 40, "shimmerSize": 40 }] }
```

`texture: "shimmer"` にすると `shimmerColor`（ラメの色）が必須になる、みたいな細かい依存もあって、ここは素直にエラーと会話しながら埋めました。

## うまくいかなかったこと

複数人が写っているシーンで、メイクが別の人の顔に乗ってしまうことが何度かありました。会議中の写真がまさにそれで、本人は素のまま、隣の同僚だけがこっそり化粧されている、という結果に。AIが「いちばんメイクを乗せやすい顔」を選ぶようで、人間が無意識に主役の顔を見るのとは逆なんですね。最終的には、その写真を本人だけに切り抜いてから渡すことで解決しました。

## シチュごとに、あえて散らす

最初は気分ごとに同じパラメータを使い回していたのですが、それだと「会議中の喜」と「通勤中の喜」が完全に同じ顔になってしまう。なので、**喜怒哀楽のジャンルは保ったまま、シチュエーションごとにリップの色やアイシャドウ、濃さを少しずつ振り分ける**ようにしました。

同じ「喜」でも、あるシーンはピンクのグロス、別のシーンはコーラル寄り、といった具合です。生成と種明かしの表示は同じ計算（`makeupFor(シーン, 気分)`）から作っているので、画面に出る数値と実際に乗せた値は必ず一致します。

## 結局、「気づける人」は何を見ているのか

自分で56枚ぶん塗ってみて、はっきり感触が出ました。気づける差と、まったく気づけない差があります。

リップの色温度と肌の血色（ツヤ）は、意外と分かります。深い赤・コーラル・くすみ、青白い肌は、小さくても「なんか違う」と感じる。一方で、眉やアイラインの強弱、ハイライトの差は、正直ほとんど分かりませんでした。

だとすると冒頭の「変化に気づくあの人」は、顔全体をぼんやり眺めているのではなくて、口元の色や肌の血色みたいな “動く・変わる一点” を無意識に見ているんじゃないか、という気がしています。観察力が高いというより、見るところを知っている、に近いのかもしれません。当たらなくても落ち込まなくて大丈夫です。たぶん、当たるほうが特殊なので。

## こんなのも面白そう、という話

作っているうちに、広げ方をいくつか思いつきました。

- **お題を当てる、人狼みたいな遊び**。「AIアートインポスター」のように、何人かが同じお題でメイクをして、ひとりだけ違うお題。どれが紛れ込んでいるかを当てる形にしても盛り上がりそうです。
- **自分の顔で参加する**。自分の写真に「今日の気分メイク」を乗せて投稿し、みんなで当て合う。あるあるは人それぞれなので、お題はいくらでも増えます。

## おわりに

バーチャルメイクは用意されたメイクから選ぶもの、という認識でしたが、色・質感・濃さ、さらに肌の血色までコードで指定できるので、「気分を顔に塗る」みたいな使い方ができました。カテゴリごとのスキーマ探しは大変でしたが、エラーと会話していくのは案外たのしかったです。
